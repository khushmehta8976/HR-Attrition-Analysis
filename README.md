# HR-Attrition-Analysis
SQL → Python → Power BI pipeline analyzing IBM HR data to identify attrition drivers and deliver actionable retention recommendations.
# Project Overview
Employee attrition is one of the most costly challenges for any organization. This project analyzes the IBM HR Analytics Employee Attrition and Performance dataset to uncover the key factors driving employee turnover and translate them into actionable retention strategies. The analysis follows a structured SQL to Python to Power BI workflow, mirroring the end-to-end pipeline used in real-world business analytics environments.
# Dataset
IBM HR Analytics Dataset from Kaggle. 1,470 employee records across 35 columns covering demographics, compensation, satisfaction scores, job role, and attrition status.
# Tools Used
MySQL for structured querying, Python with Pandas and Seaborn for data cleaning and EDA, and Power BI for the interactive dashboard.
# Key Findings
Overall attrition rate was 16.1%, above the industry average. Sales had the highest attrition at 20.6%. Employees working overtime were 2.5 times more likely to leave. Employees with under 2 years tenure accounted for over 35% of all attrition. A counterintuitive finding: higher salary band employees showed disproportionately high attrition, pointing to a career growth ceiling rather than a pay problem.
# Recommendations
Introduce structured onboarding for new hires in the first 90 days. Audit overtime practices in the Sales department. Review career progression paths for senior employees. Use job satisfaction scores of 1 to 2 as early warning signals for manager intervention.
